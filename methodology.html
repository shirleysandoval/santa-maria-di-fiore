Methodology

This project was developed following a structured process to organize and represent information about the Cattedrale di Santa Maria del Fiore in a clear and meaningful way. The first step involved collecting reliable information about the cathedral, focusing on its history, architecture, and cultural significance. After gathering the data, the information was organized into semantic categories using RDF principles, where knowledge is represented in the form of subject–predicate–object relationships. This structure allowed us to define clear connections between entities such as the cathedral, its architect Filippo Brunelleschi, and its location in Florence. Once the data model was defined, SPARQL queries were created to demonstrate how information could be retrieved efficiently from the dataset. These queries were designed to extract key elements such as the architect, location, historical context, and architectural style. Finally, the website was developed using HTML and CSS, creating a multi-page structure with a navigation menu to connect all sections. The project was then deployed using GitHub Pages, making it accessible online as a complete digital presentation of the cathedral.
